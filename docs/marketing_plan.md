## 2. ハンドメイドサイト戦略（minne / Creema）

### 方針
minne・Creemaで「レジンタグ」「実物カード」を出品し、**アプリDL促進** と **グッズ売上** の両方を狙う。
2パターンの商品を併売し、どちらが売れるか実験する。

### パターンA：セルフプラン（安い）

| 項目 | 内容 |
|------|------|
| 商品名例 | 「うちの子免許証レジンタグ【アプリで自作】」 |
| 価格 | 現行グッズ価格そのまま |
| 流れ | 購入 → アプリDL案内 → ユーザーが免許証作成 → データ送信 → しゅーとがグッズ製作 |
| メリット | しゅーとの作業が少ない、アプリDL数も増える |
| デメリット | ユーザー側の手順が多い（アプリDL→作成→データ送信） |

### パターンB：おまかせプラン（高い）

| 項目 | 内容 |
|------|------|
| 商品名例 | 「うちの子免許証レジンタグ【写真送るだけ！デザインおまかせ】」 |
| 価格 | +500〜1,000円上乗せ（デザイン作成料として） |
| 流れ | 購入 → 写真とペット情報をメッセージで受け取る → しゅーとがデータ作成 → グッズ製作 |
| メリット | アプリが苦手な人も買える、客単価が高い |
| デメリット | しゅーとのデザイン作成作業が発生する |

### 出品時のポイント
- 商品説明欄にアプリのダウンロードリンクを記載（パターンA）
- ハンドメイドサイトの客層は「写真送るだけ」の手軽さを好む → **パターンBの方が売れやすい可能性**
- 両パターン出品して売れ行きを比較

## 3. SNSプロモーション

### 3.1 TikTok動画企画

スマホ1台で撮れる、編集ほぼ不要なもの。週2〜3本ペースで継続投稿が理想。

| # | 動画案 | 秒数 | 撮り方 |
|---|--------|------|--------|
| 1 | 「うちの子に免許証つくってみた」 | 15秒 | アプリ画面録画→完成品ドーン。王道 |
| 2 | 「免許証の写真撮影に協力してくれない犬/猫」 | 15秒 | 正面写真を撮ろうとするペットの様子→できた免許証 |
| 3 | 「うちの子のコスチューム全部試してみた」 | 30秒 | 画面録画でコスチュームをポンポン切り替え |
| 4 | 「免許証とうちの子を並べてみた」 | 10秒 | 完成した免許証とペットを横に並べるだけ |
| 5 | 「迷子札にNFCで情報書き込んでみた」 | 15秒 | レジンタグにスマホをかざす→情報が表示される |
| 6 | 「実物カード届いた！開封」 | 15秒 | グッズが届く→開封→ペットの首輪につける |
| 7 | 「友達のペットの免許証こっそり作ってプレゼントした」 | 20秒 | 写真もらう→作成→渡す→リアクション |
| 8 | 「うちの子の証明写真が盛れすぎた」 | 10秒 | 元写真とAI背景切り抜き後の比較 |
| 9 | 「多頭飼いの免許証コレクション」 | 15秒 | 複数のペットの免許証を1枚ずつ見せていく |
| 10 | 「このアプリ作ったの実は俺です」 | 15秒 | 普通に免許証紹介→最後に「開発者は自分」のオチ |

**特におすすめ：2, 6, 10**
- 2 → ペットの「言うこと聞かない感」は鉄板で再生数伸びる
- 6 → 開封動画は物欲刺激＋グッズ販売に直結
- 10 → 「個人開発者」ネタはTikTokで共感されやすい

### 3.2 ハッシュタグ戦略

- 固定タグ：`#うちの子免許証` `#ペット免許証`
- 追加タグ例：`#犬のいる暮らし` `#猫のいる暮らし` `#ペット` `#個人開発`
- アプリのシェア機能にハッシュタグを自動付与（要実装確認）

### 3.3 その他SNS施策

- **Twitter/X**：開発者アカウントで自分のペットの免許証を投稿＋開発ストーリー
- **Instagram**：完成品の高品質な写真を投稿（映えるコスチューム＋かわいいペット）
- **ペット系マイクロインフルエンサー**（フォロワー1,000〜10,000人）にDM。RevenueCatのPromo機能でプレミアムを付与して使ってもらう

### 4.4 キーワード選定の考え方
- アプリ名・サブタイトルに入れた語とキーワードフィールドは**完全重複ゼロ**が原則
- 動物種名でロングテール検索をカバー
- 「コスプレ」「着せ替え」→ コスチューム機能の検索
- 「コラージュ」「加工」「証明写真」→ 写真加工アプリを探す人
- 「SNS」「ネタ」→ シェア目的のユーザー
- 「プレゼント」「記念」「誕生日」→ イベント需要
- 月1回 App Store Connect のデータを見てキーワードを見直す

### 4.5 スクリーンショット方針
- 最初の2枚で「何ができるアプリか」を伝える（Before→After形式が効果的）
- 犬種・猫種の人気作例を多く見せる
- **なめ猫連想回避**：猫＋学ラン＋免許証の組み合わせはメインビジュアルに使わない
- 犬＋サングラス等を前面に出す

## 5. 課金・収益戦略

### 5.1 現行モデルの評価
- **¥300買い切りは適切**（ペットアプリ課金相場の範囲内、ワンコインで迷わず買える）
- 値上げより**課金率を上げる**方が優先

### 5.2 構造的課題
- LTV（顧客生涯価値）の上限が¥300で固定される
- 一度買ったユーザーからの追加収益がグッズ以外にない

### 5.3 追加収益レイヤーの方向性
1. **コスチュームパック販売**：季節限定・テーマ別を¥120〜200/パックで個別販売（design_document.md Phase B参照）
2. **グッズ販売の拡大**：ハンドメイドサイトへの展開（本ファイルセクション2参照）
3. **グッズ注文プロセスの信頼性向上**：Googleフォーム → BASE/STORES等への移行検討

### 5.4 課金率を上げるUX施策
- 無料2枚目作成後に「プレミアムコスチュームプレビュー画面」を挟む
- プレミアムコスチュームの「試着」機能（透かし付きプレビュー→保存はプレミアム必要）
- 価格アンカリング：「うちの子のおやつ1個分でプレミアム！」
