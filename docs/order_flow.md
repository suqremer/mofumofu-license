# 注文〜発送フロー

## 全体の流れ

```
お客さん: Stripe決済 → Googleフォーム送信
    ↓
しゅーと: メール通知受信 → 管理シート確認
    ↓
しゅーと: 製造（カード印刷 / タグ製作）
    ↓
（NFC代行ありの場合）
しゅーと: NTAG215に書き込み → Stripe請求書で+¥500請求 → 入金確認
    ↓
しゅーと: 梱包 → 発送 → ステータスを「発送済」に変更
    ↓
自動: 発送日記入 + 会計スプレッドシートに売上連携
```

## 詳細手順

### 1. 注文受付
- お客さんがアプリ内の「注文する」からStripe Payment Linkで決済
- 決済後、アプリ内の案内に従ってGoogleフォームで画像+情報を送信
- **自動処理**: 管理番号が採番（#001〜）、ステータス「未対応」、NFC代行の判定
- **自動処理**: しゅーとにメール通知（注文内容+NFC代行有無）

### 2. 注文確認
- Stripeダッシュボードで決済を確認
- 管理シートでフォーム回答を確認
- Stripe決済とフォーム回答を突き合わせ（お名前+日時で照合）

### 3. 製造
- ステータスを「製造中」に変更
- カード: PVC印刷
- タグ: レジン製作
- セット: 両方製造

### 4. NFC書き込み代行（該当注文のみ）
- 管理シートでNFC請求列が「未請求」の行を確認（黄色ハイライト）
- NTAG215タグにお客さんの指定内容を書き込み
- Stripeダッシュボード → 請求書作成 → お客さんのメールに+¥500請求
- NFC請求列を「請求済」に変更
- 入金確認後「入金済」に変更
- **自動処理**: 毎日9時にNFC未請求のリマインダーメール

### 5. 発送
- プチプチで梱包
- 普通郵便（定形外）で発送
- ステータスを「発送済」に変更
- **自動処理**: 発送日が自動入力
- **自動処理**: 会計スプレッドシートの売上シートに自動記入

### 6. 完了
- 配達完了後、ステータスを「完了」に変更

## 管理シートの列

### フォーム回答（A〜H列、自動）
| 列 | 内容 |
|---|------|
| A | タイムスタンプ |
| B | お名前 |
| C | 商品種別 |
| D | 1デザインあたりの枚数 |
| E | 印刷する免許証の画像 |
| F | NFC書き込み代行 |
| G | NFC書き込み内容 |
| H | 備考 |

### 管理列（I〜O列、しゅーと管理）
| 列 | 内容 | 入力方法 |
|---|------|----------|
| I | 管理番号（#001〜） | 自動 |
| J | ステータス | ドロップダウン |
| K | NFC請求 | ドロップダウン |
| L | 発送日 | 自動（発送済にしたら） |
| M | 追跡番号 | 手動（普通郵便なら空欄） |
| N | メモ | 手動 |
| O | 会計連携 | 自動 |

### ステータスの流れ
```
未対応 → 製造中 → 製造完了 → 発送済 → 完了
                                    ↘ キャンセル
```

## 発送方法
- 普通郵便（定形外）: 120〜140円
- プチプチで梱包
- 追跡番号なし（レターパックライトに変更する場合は370円）
